# 子技能：伏笔台账模板

> 本文件对应主技能 SKILL.md §11.2，用于管理 `foreshadowing_ledger.md` 中的伏笔条目。

## 用途

当 Agent 需要新增、修改或检索伏笔台账条目时，加载本文件获取模板规范。
伏笔台账文件路径：`{project}/02_memory/foreshadowing_ledger.md`

---

## 编号规则

- 格式：`F` + 三位数字，例如 `F001`、`F012`
- 新增伏笔时取当前最大编号 + 1
- 编号一旦分配，不得更改（即使伏笔废弃）

---

## 状态值规范

`当前状态` 字段只能从以下值中选择：

| 状态值   | 含义                         |
| -------- | ---------------------------- |
| 未埋设   | 已规划但尚未写入正文         |
| 已埋设   | 已出现在正文中               |
| 持续发酵 | 已埋设，仍在正文中持续作用   |
| 部分揭示 | 真相已有部分透露             |
| 已回收   | 真相已在正文中完整揭示       |
| 废弃     | 决定不再使用此伏笔           |
| 需要修改 | 内容与当前剧情冲突，需要修订 |

---

## 模板格式

```markdown
## 伏笔编号：[编号，如 F001, F002]

### 1. 伏笔名称【必填】

一句话命名这个伏笔。
（例如：丹炉灰烬中的指骨）

---

### 2. 伏笔内容【必填】

读者在正文中看到的表层信息。
（例如：主角在清理丹炉灰烬时，发现一截没有烧毁的指骨，上面刻着模糊的符号。）

---

### 3. 埋设章节【必填】

这个伏笔第一次出现在哪一章。
（例如：第 1 章）

---

### 4. 表层解释 / 误导方向【必填】

读者或主角当时会以为这是什么。
（例如：读者会以为这是丹阳子炼丹失败后留下的普通尸骨，主角也暂时认为这是被害弟子的遗物。）

---

### 5. 真实含义【必填】

这个伏笔真正代表什么。
（例如：这截指骨属于上一任拥有特殊感知能力的弟子，符号其实是身份标记，说明丹阳子一直在筛选同类体质的人。）

---

### 6. 计划回收章节【推荐】

准备在哪一章揭示真相。
（例如：第 6 章）

---

### 7. 回收方式【推荐】

真相如何被揭开。
（例如：主角在密室中发现多具刻有相同符号的遗骨，意识到自己不是第一个被选中的人。）

---

### 8. 当前状态【必填】

伏笔现在处于什么阶段（从上方状态值表中选择）。
```

---

## 填写规则

1. 字段 1、2、3、4、5、8 为**必填**，不得留空。
2. 字段 6、7 为推荐项，规划阶段可暂填"待定"，但在埋设后必须补全。
3. 每个字段块之间以 `---` 分隔。
4. 状态值必须严格使用上方状态值表中的词，不得自行创造新状态。
5. 若伏笔内容已变化（如改变回收方式），应更新对应字段并记录修改日期（可在末尾追加 `> 更新于 YYYY-MM-DD：原因`）。

---

## 与 GUI 的关系

`foreshadowing_ledger.md` 由 GUI「伏笔台账」面板直接读写。  
GUI 卡片中的字段对应本模板的字段编号（1-8），额外自定义字段从第 9 号起追加。  
Agent 手动编辑文件时，必须保持本模板格式，否则 GUI 解析可能失败。
