# 子技能：章节大纲模板

> 本文件对应主技能 SKILL.md §11.4，用于规划和填写 `novel_directory.md` 中的章节蓝图条目。

## 用途

当 Agent 需要生成、补全或更新章节大纲时，加载本文件获取模板规范。  
章节大纲文件路径：`{project}/02_memory/novel_directory.md`

---

## 与 GUI 的关系

`novel_directory.md` 由 GUI「章节目录」面板直接读写。  
GUI 卡片中的 6 个字段与本模板一一对应，保存时以 `字段名：内容` 单行格式序列化写回文件。  
Agent 手动编辑文件时，**必须保持单行格式**（每个字段占一行，不换行），否则 GUI 解析会将内容截断。

---

## 章节标题格式

```text
### 第 {N} 章 {标题}
```

---

## 6 个必填字段

### 1. 本章概括【必填】

用 **1 句话** 说明本章发生了什么，包含关键转折或结果。

格式：

```text
本章概括：[一句话概括]
```

示例：

```text
本章概括：主角第一次进入丹房，被迫协助炼丹，并发现所谓"仙药"来自活人身体。
```

---

### 2. 剧情目标【必填】

说明本章在全书中承担的叙事职责，可多选以下类型（用分号分隔）：

- 建立人物 / 建立世界
- 推进主线 / 推进支线
- 埋设伏笔 / 回收伏笔
- 制造冲突 / 升级冲突
- 角色成长 / 关系转变
- 信息揭示 / 误导读者
- 阶段性高潮 / 情绪释放
- 引出新地图 / 新势力

格式：

```text
剧情目标：[类型，分号分隔]
```

示例：

```text
剧情目标：推进主线；埋设伏笔；制造冲突
```

---

### 3. 开头设计【必填】

说明本章如何开场。建议包含：开场地点与氛围、开场人物与状态、第一个钩子。

格式：

```text
开头设计：[开场描述]
```

示例：

```text
开头设计：从主角在潮湿昏暗的丹房中磨药开始，先写环境压迫感，再通过师傅突然点名，让主角意识到自己可能成为下一个药引。
```

---

### 4. 核心冲突【必填】

本章最主要的矛盾。建议结构：**主角想要什么 → 阻碍来自哪里 → 冲突如何升级或爆发**。

格式：

```text
核心冲突：[冲突描述]
```

示例：

```text
核心冲突：主角想要隐藏已发现真相 → 阻碍来自师傅的试探和弟子监视 → 冲突升级为师傅要求主角亲手处理一具"药材"。
```

---

### 5. 关键情节节点【必填】

3 到 5 个核心剧情节点，按时序排列，不要超过 5 个。

格式：

```text
关键情节节点：1. … 2. … 3. … 4. … 5. …
```

示例：

```text
关键情节节点：1. 师傅召集全体弟子进入惠安殿 2. 主角亲眼目睹黑鼎吞噬逃跑者 3. 师傅当众提拔主角并赐道号 4. 主角私下清点药引数量发现异常 5. 主角夜间独自观察黑鼎洞口
```

---

### 6. 结尾设计【必填】

本章如何收束并制造下一章的阅读驱动力。建议包含：情绪落点（如恐惧、释然、震惊）、结尾方式（悬念 / 反转 / 成功 / 失败 / 信息揭示）、对下一章的牵引。

格式：

```text
结尾设计：[结尾描述]
```

示例：

```text
结尾设计：情绪落点为震惊与动摇；结尾方式为信息揭示；主角离开惠安殿时首次动摇"这世界是否真实"，为下章正式开始怀疑埋下种子。
```

---

## 完整示例

```markdown
### 第 1 章 师傅

本章概括：李火旺在诡异地下道观中捣药、出手护人、被师傅召见，亲眼目睹师傅将痴傻师姐捣入丹炉炼丹，章末被揭示为精神病院的束缚患者，整个场景是幻觉。

剧情目标：建立人物；建立世界；制造冲突；埋设伏笔；信息揭示

开头设计：开场于阴暗潮湿的地下溶洞料房，以李火旺百无聊赖捣药的动作起手，营造压抑封闭环境；胖子欺辱白化病少女提供第一个冲突钩子，引出主角性格与地位。

核心冲突：李火旺想压制情绪、保持冷漠隐忍 → 阻碍来自丑陋凶残的师傅与料房内无处不在的恶意 → 冲突在目睹师傅将活人打入丹炉后达到最高张力，主角被迫直面这个世界的残酷真实性。

关键情节节点：1. 胖子欺辱少女，主角砸人护人 2. 青衣道士传召，白化病少女恐惧阻拦 3. 途经地下道观，痴傻师姐给主角稀糖 4. 师傅当众将痴傻师姐捣入石瓮炼丹 5. 师傅质问骂师一事，主角内心默念后猛睁眼——切换至病房

结尾设计：情绪落点为真相撕裂式震惊；结尾方式为硬反转式信息揭示；道观幻境消失病房出现，读者与主角同步意识到第一章是精神病幻觉，牵引下一章追问哪条线才是真实。
```

---

## 填写规则

1. 6 个字段全部**必填**，不得省略。
2. 每个字段值必须写在**同一行**内（不换行），GUI 解析依赖单行格式。
3. `关键情节节点` 中各节点用 `1. 2. 3.` 序号连接在一行内。
4. 若大纲过于粗糙（缺少 3 个以上字段），Agent 应主动提示补强，而不是直接进入正文写作。
5. 修改某章大纲后，检查是否需要同步更新 `novel_settings.md`、`character_state.md` 或 `foreshadowing_ledger.md`。
