# 子技能：出场角色节点模板

> 本文件对应主技能 SKILL.md §11.1，用于填写和更新 `plot_arcs.json` 中每个 `PlotBeat` 条目。

## 用途

当 Agent 需要填写、扩展或更新 `plot_arcs.json` 的 beat 数据时，加载本文件获取模板规范。

---

## 字段映射说明

| 模板字段             | 对应 JSON 键  | 说明                         |
| -------------------- | ------------- | ---------------------------- |
| 角色                 | `character`   | 出场角色姓名                 |
| 本章身份 / 当前处境  | `event` 首行  | 公开身份、地理位置、心理负担 |
| 本章目标             | `event`（续） | 核心行动诉求（明线或暗线）   |
| 本章行动             | `event`（续） | 具体关键行动、核心剧情点     |
| 与其他角色的关系变化 | `event`（续） | 本章结束后的关系漂移         |
| 本章结束状态         | `event`（续） | 当章最后一幕的处境           |

`event` 字段将上述五个字段**合并**写入，各字段之间用换行分隔，格式如下：

---

## 模板格式

```text
角色：[人物名称]
本章身份 / 当前处境：[描述]
本章目标：[目标]
本章行动：[行动]
与其他角色的关系变化：[关系漂移]
本章结束状态：[状态]
```

---

## JSON 示例

```json
{
  "character": "李火旺",
  "event": "本章身份 / 当前处境：清风观最低等药引，被师傅监视\n本章目标：偷取玄阳玉佩并带回现实\n本章行动：深夜潜入玄阳洞室得手，撞见玄阳带人逃跑\n与其他角色的关系变化：对玄阳由竞争转为短暂同情，对丹阳子戒备加深\n本章结束状态：持玉佩安全离开，半信半疑幻境是否为真",
  "scene_id": "s0"
}
```

---

## 填写规则

1. `character` 与 `event` 均为必填。
2. `event` 中五个子字段必须全部填写，不得省略任何一个字段标签（即便内容为"无变化"也要写明）。
3. 子字段之间使用 `\n` 换行，不使用空行。
4. 每个 beat 必须绑定到至少一个 `scene_id`，scene_id 与 `plot_arcs.json` 场景节点对应。
5. 一章中可出现多个 beat（多个出场角色各自一条）。
